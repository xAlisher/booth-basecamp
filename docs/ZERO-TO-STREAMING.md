# Zero to Streaming — run your own station, headless, that survives reboots

> **Audience:** a peer (or an AI agent acting for one) who wants to run their **own** radio station
> with their **own** content on a Linux box, exactly the way we run **Parallel Society Radio (PSR)**
> on our server "Sneg": a **headless** Basecamp under a virtual display, a **Booth** (`radio_module`)
> origin that spawns MediaMTX + Tor, fed by **Liquidsoap** as an automated DJ, all wired to
> **`systemd --user`** so it comes back after a reboot.
>
> This guide is **agent-executable**: every step is a copy-pasteable block. It is **grounded in the
> code** (`radio_module/src/radio_plugin.cpp`) and in the **real, running PSR units** on Sneg — the
> systemd units, `run-app.sh`, and `station.liq` below are the sanitized originals, not invented.
>
> **Sibling docs (read/reuse, this guide links them):**
> [`CONNECTING-LIQUIDSOAP.md`](CONNECTING-LIQUIDSOAP.md) (the feed half, in depth) ·
> [`parallel-society-radio/station.liq`](parallel-society-radio/station.liq) (our real playout script) ·
> [`CONNECTING-OBS.md`](CONNECTING-OBS.md) (live-capture alternative to Liquidsoap) ·

## 0. The picture (and the one idea that makes "headless" work)

```
  ┌───────────────────────── your Linux box (e.g. Debian/Ubuntu) ─────────────────────────┐
  │                                                                                       │
  │  Xvfb :99  ◄── virtual display (no monitor needed)                                    │
  │     ▲                                                                                 │
  │  Basecamp AppImage (headless)                                                         │
  │     └─ Booth = radio_module (core)  ──spawns──►  MediaMTX (origin)  ──spawns──►  Tor  │
  │           │  reads station.json on load → AUTO-RESUMES the station                    │
  │           │  announces over LogosMessaging every 15s (TTL 45s)                        │
  │           ▼                                                                           │
  │        RTMP :1935  ◄──────────────────  Liquidsoap (automated DJ)                     │
  │                                          plays your normalized folders, pushes AAC    │
  │        HLS :8888  ──► onion :80 ──────────────────────────────────────────────┐       │
  └───────────────────────────────────────────────────────────────────────────────┼───────┘
                                                                                  ▼
                            a listener's  Receiver  discovers over LogosMessaging, verifies the
                            station's signature, and plays the HLS stream over Tor
```

**The key idea — auto-resume.** You never script "click Start" on every boot. You configure a station
**once**; Booth persists it to `station.json` with `"running": true`. On **every** Basecamp launch,
`radio_module` reads that file and **re-spawns MediaMTX + Tor and re-announces automatically**
(`radio_plugin.cpp:431`). So "survive a reboot" reduces to "keep the Basecamp app alive under
systemd" — the station rides along.

**One caveat, by design:** a **Keycard**-identity station does **NOT** auto-resume — the signing key
can't be re-derived at boot with no card present, so resuming would broadcast under a wrong/unsigned
identity (`radio_plugin.cpp:447-451`). For an unattended box use the **Autogenerated** (device-local
key) identity, which is what PSR uses (`"keySource": "autogen"`). See [§7](#7-what-persists-what-doesnt).

## 1. Prerequisites — prepare the box

Assumed: a **Debian/Ubuntu-like** Linux box you control. Adjust package names for
other distros. Install the toolchain:

```bash
sudo apt update
sudo apt install -y \
  liquidsoap ffmpeg \        # automated DJ + transcode/loudness/probe
  xvfb xdotool x11-utils \   # virtual display + window automation (xdpyinfo/xdotool)
  tor \                      # onion privacy (Booth also bundles its own tor; system tor is a fallback)
  python3 yt-dlp \           # ripping + small JSON helpers
  loudgain                   # ReplayGain loudness tagging (or use ffmpeg loudnorm, §3)

# MediaMTX is not in apt. Install a static build to ~/.local/bin (Booth finds it via RADIO_MEDIAMTX_BIN):
mkdir -p ~/.local/bin
# either:  nix profile install nixpkgs#mediamtx   (then symlink into ~/.local/bin)
# or grab a release binary from https://github.com/bluenviron/mediamtx/releases and:
#   tar xf mediamtx_*_linux_amd64.tar.gz -C ~/.local/bin mediamtx
~/.local/bin/mediamtx --version || echo "install mediamtx before continuing"

# sanity
liquidsoap --version   # MUST have the %ffmpeg/AAC encoder (see CONNECTING-LIQUIDSOAP.md §1 if not)
ffmpeg -hide_banner -encoders | grep -q aac && echo "aac ok"
```

> Booth's first-launch **dependency preflight** (v0.2.1, #53) will also flag any missing
> `mediamtx`/`tor`/`ffmpeg` when the panel opens — but on a headless box you won't see the card, so
> install them up front here.

You also need a **Basecamp AppImage with Booth installed** (`radio_module` + `radio_ui`). Grab the
signed release and install per [`../README.md#install-into-logos-basecamp`](../README.md). Keep the
AppImage path handy (PSR uses `~/logos-basecamp-radio-268.AppImage`).

## 2. Get your content — rip → transcode → **normalize** → tag → lay out

This is the part no other doc covers. The goal: a set of **loudness-consistent, tagged** audio files
in a **folder-per-role** layout that Liquidsoap can rotate.

### 2.1 Folder layout (roles = rotation weights later)
```bash
LIB=/mnt/music/parallel-society-radio     # pick your own path; keep it off the OS disk if you can
mkdir -p "$LIB"/{talks,dj,jingles}
# talks/   = spoken word / conference recordings   (quiet, ~-16 LUFS source)
# dj/      = music sets                              (loud, ~-6 LUFS source)  ← the ~10 LUFS gap problem
# jingles/ = short station-ID / interstitials        (your "intents")
```

### 2.2 Rip (example: our anchor was youtube.com/@Logos_network)
```bash
# a single video → best audio, no video track:
yt-dlp -x --audio-format flac --audio-quality 0 \
  -o "$LIB/talks/%(title)s.%(ext)s" "<youtube-url>"

# a whole playlist/channel of DJ sets:
yt-dlp -x --audio-format flac -o "$LIB/dj/%(title)s.%(ext)s" "<playlist-url>"
```
> **Format rule.** Keep **FLAC where the source is lossless**, else **320k MP3**. Playout re-encodes to
> **AAC 128k** at the end — so avoid lossy-on-lossy (don't rip to 128k MP3 then re-encode). Liquidsoap
> reads `.flac/.mp3/.m4a/.ogg`.

### 2.3 Normalize loudness — the talks-vs-DJ ~10 LUFS gap
Talks land near **−16 LUFS**, music near **−6 LUFS**; without normalization a listener rides the volume
knob across every transition. **Normalize offline, at rip time** — best quality, and it means the light
playout limiter (§4) rarely has to work.

**Option A — ReplayGain tags (recommended; non-destructive, leaves audio untouched):**
```bash
# tags each file with the gain needed to hit the target; Liquidsoap applies it at playout (§4).
loudgain -a -k -s e "$LIB"/talks/*.flac    # -a = album mode off per-track; -k = clip-safe; -s e = write tags
loudgain -a -k -s e "$LIB"/dj/*.flac
```

**Option B — two-pass `ffmpeg loudnorm` (destructive; bakes the level in):** use when a player/encoder
won't honor ReplayGain. Target **talks −16 LUFS**, **music −14 LUFS**:
```bash
normalize() {  # $1=infile  $2=outfile  $3=target LUFS
  # pass 1: measure
  data=$(ffmpeg -hide_banner -i "$1" -af loudnorm=I=$3:TP=-1.5:LRA=11:print_format=json -f null - 2>&1 \
         | python3 -c 'import sys,json,re;t=sys.stdin.read();j=json.loads(t[t.rfind("{"):t.rfind("}")+1]);print(j["input_i"],j["input_tp"],j["input_lra"],j["input_thresh"])')
  read I TP LRA THR <<<"$data"
  # pass 2: apply, measured
  ffmpeg -hide_banner -y -i "$1" \
    -af loudnorm=I=$3:TP=-1.5:LRA=11:measured_I=$I:measured_TP=$TP:measured_LRA=$LRA:measured_thresh=$THR:linear=true \
    -c:a flac "$2"
}
# normalize "$LIB/talks/foo.flac" "$LIB/talks/foo.norm.flac" -16
```
> **Avoid** Liquidsoap's adaptive `normalize()` as the *primary* tool — it pumps on music. Do the heavy
> lifting offline (A or B); use only the light `limit()`/`enable_replaygain()` at playout as a net.

### 2.4 Tag metadata (drives now-playing)
Booth puts **`artist — title`** into the announce (#35), read from a `nowplaying.txt` Liquidsoap writes.
So make sure files carry `artist`/`title` tags:
```bash
# FLAC:
metaflac --set-tag=ARTIST="Speaker Name" --set-tag=TITLE="Talk Title" "$LIB/talks/foo.flac"
# MP3 (mid3v2 from python3-mutagen):
mid3v2 -a "Artist" -t "Title" "$LIB/dj/bar.mp3"
# yt-dlp can also embed at rip time: add  --embed-metadata
```

## 3. Program the station — build your `station.liq`

Start from our real, proven playout script:
[`parallel-society-radio/station.liq`](parallel-society-radio/station.liq). It does exactly what a
station needs: **watched folders**, **weighted rotation** (more music than talk), a **jingle every ~6
items** (your "intents"), **crossfades that carry metadata**, and a **now-playing file**. Annotated:

```liquidsoap
# station.liq — playout brain. Launch: PSR_RTMP_PATH=<path> PSR_RTMP_PASS=<pass> liquidsoap station.liq
settings.server.telnet.set(true)          # optional: telnet control on :1234 (skip/metadata)
settings.server.telnet.port.set(1234)

lib = "/mnt/music/parallel-society-radio"

# --- sources: one watched folder per role. reload_mode="watch" = drop files in live, no restart -----
talks   = playlist(mode="randomize", reload_mode="watch", "#{lib}/talks")
djsets  = playlist(mode="randomize", reload_mode="watch", "#{lib}/dj")
jingles = playlist(mode="randomize", reload_mode="watch", "#{lib}/jingles")

# stamp jingles so the announce reads a friendly "artist — title" during them (#32/#35)
def stamp_jingle(_m) = [("artist", "Your Station"), ("title", "ID")] end
jingles = metadata.map(update=true, strip=false, insert_missing=true, stamp_jingle, jingles)

# --- programme: 1 talk : 2 DJ, then a jingle roughly every 6 items (tune the weights) --------------
base      = rotate(weights=[1, 2], [talks, djsets])
programme = rotate(weights=[1, 5], [jingles, base])

# --- LOUDNESS NET at playout (light — the heavy lifting was done offline in §2.3) -------------------
# radio = enable_replaygain(programme)   # uncomment if you used loudgain (Option A) — applies the tags
radio = crossfade(fade_in=1., fade_out=1., duration=3., programme)  # smooth + propagates metadata
radio = mksafe(radio)                     # never let the source go silent/undefined
radio = limit(radio)                      # gentle brickwall safety net

# --- now-playing → file that Booth reads into the announce (#35) ------------------------------------
def on_meta(m) =
  line = if m["artist"] != "" then "#{m["artist"]} — #{m["title"]}" else m["title"] end
  ignore(file.write(data=line, "#{lib}/nowplaying.txt"))
end
radio = source.on_metadata(radio, on_meta)

# --- output: AAC-in-FLV → Booth's MediaMTX RTMP ingest ---------------------------------------------
rtmp_path = environment.get(default="", "PSR_RTMP_PATH")   # from Booth's stream card (before the '?')
rtmp_pass = environment.get(default="", "PSR_RTMP_PASS")   # the pass= value
rtmp_url  = "rtmp://127.0.0.1:1935/#{rtmp_path}?user=publisher&pass=#{rtmp_pass}"
output.url(
  url = rtmp_url,
  %ffmpeg(format="flv", %audio(codec="aac", b="128k", samplerate=44100, channels=2)),  # match Booth's ingest
  radio)
```

Point `lib` at your `$LIB`, tune the `rotate` weights, drop your jingles in `jingles/`. Keep the AAC
target **128k / 44.1 kHz / stereo** — that's what the ingest expects. Deeper reference + a single-folder
minimal variant: [`CONNECTING-LIQUIDSOAP.md`](CONNECTING-LIQUIDSOAP.md).

## 4. Mint the station **once** (so it can auto-resume forever)

Booth auto-resumes whatever is in `station.json` with `"running": true`. You create that file once by
starting a station. Two ways:

**Easiest — set it up with a visible display once.** Run Booth normally (on a desktop, or VNC into the
Xvfb `:99` display you'll set up in §5), then in the panel: **name** the station → **Autogenerated**
identity (NOT Keycard, for unattended boxes) → **Public** reach → **Tor** privacy → **Start**. Booth
mints `station.json` and shows the **stream card** (RTMP path + key). Grab both.

The persisted file looks like this (real PSR, secret redacted) — its schema is your source of truth:
```json
{
  "name": "Parallel Society Radio",
  "description": "Parallel Society festival — talks + DJ sets (@Logos_network)",
  "path": "33a5971eeba1d06a",            // public stream path (the part before '?' on the card)
  "streamKey": "REDACTED",               // secret publish pass — treat like a password
  "privacy": "onion",                    // "onion" (Tor) or "lan"
  "visibility": "public",                // "public" directory topic, or a private per-stream topic
  "keySource": "autogen",                // autogen = auto-resumes · keycard = will NOT auto-resume
  "hostLabel": "anonymous",
  "announceTopic": "/radio-basecamp/1/directory/json",
  "running": true,                       // ← the auto-resume flag. Stop sets this false.
  "startedAt": 1781176782050
}
```
It lives at (respects `XDG_DATA_HOME`, `radio_plugin.cpp:375`):
```
$XDG_DATA_HOME/radio_module/station.json
# on Sneg, with XDG_DATA_HOME=$HOME/.local/share/Logos-radio →
#   ~/.local/share/Logos-radio/radio_module/station.json
```
**Feed the `path` + `streamKey` into your Liquidsoap launch** (`PSR_RTMP_PATH` / `PSR_RTMP_PASS`, §3/§5).

## 5. Install as `systemd --user` services — survive reboot

This is the durability layer, and these are the **actual units running PSR on Sneg** (paths/secrets
adjusted to placeholders). Four concerns → three units + a helper script. Liveness is tied together with
`After=`/`Wants=` so they start in order and each restarts on its own.

### 5.1 The app launcher — `~/radio-station/run-app.sh`
Launches the headless AppImage on display `:99`, waits for core boot (~80 s), opens the radio panel so
`radio_module` loads (which triggers **auto-resume**), and only *falls back* to a Start click if the
broadcast didn't come up. Ties its lifetime to the app so systemd restarts the whole thing on crash.

```bash
#!/usr/bin/env bash
set -uo pipefail
cd "$HOME"
export DISPLAY=:99
export QT_QUICK_BACKEND=software        # headless GPU-less rendering
export LIBGL_ALWAYS_SOFTWARE=1
export XDG_DATA_HOME="$HOME/.local/share/Logos-radio"      # → where station.json lives (§4)
export RADIO_MEDIAMTX_BIN="$HOME/.local/bin/mediamtx"       # tell Booth where MediaMTX is (radio_plugin.cpp:200)
export PATH="$HOME/.local/bin:/usr/bin:/bin:$PATH"
export RADIO_NOWPLAYING_FILE="/mnt/music/parallel-society-radio/nowplaying.txt"  # now-playing source (radio_plugin.cpp:70)
APPIMG="$HOME/logos-basecamp-radio-268.AppImage"           # your Booth-enabled Basecamp AppImage
LOG="$HOME/radio-station/app.log"

# wait for Xvfb :99
for i in $(seq 1 30); do xdpyinfo -display :99 >/dev/null 2>&1 && break; sleep 1; done

"$APPIMG" >"$LOG" 2>&1 &
APP=$!

# wait for the window, then for core boot (~80s total on Sneg)
for i in $(seq 1 90); do xdotool search --name "Logos App" >/dev/null 2>&1 && break; sleep 2; done
sleep 70

# open the radio panel → loads radio_module → auto-resumes the "running": true station
xdotool mousemove 33 334 click 1 >/dev/null 2>&1
sleep 25

# fallback: if the broadcast didn't come up on its own, press Start
if ! pgrep -x mediamtx >/dev/null 2>&1; then
  xdotool mousemove 125 445 click 1 >/dev/null 2>&1
  sleep 15
fi

wait "$APP"     # app exits → script exits → systemd restarts us
```
> ⚠️ **The two `xdotool mousemove … click` lines are coordinate-based** (panel icon at `33,334`; Start
> at `125,445` for a `1600x1000` display). They are the fragile part. If your build/resolution differs,
> VNC into `:99` once, find the real coordinates, and update them. In the happy path auto-resume fires
> and the Start click is never needed — the coordinates only matter for the fallback and the initial
> panel open.

```bash
chmod +x ~/radio-station/run-app.sh
```

### 5.2 Unit 1 — virtual display `~/.config/systemd/user/logos-radio-xvfb.service`
```ini
[Unit]
Description=Logos Radio - Xvfb virtual display :99
After=network-online.target

[Service]
Type=simple
ExecStart=/usr/bin/Xvfb :99 -screen 0 1600x1000x24
Restart=always
RestartSec=3

[Install]
WantedBy=default.target
```

### 5.3 Unit 2 — the app `~/.config/systemd/user/logos-radio-app.service`
```ini
[Unit]
Description=Logos Radio - Basecamp app (headless, station auto-resume)
After=logos-radio-xvfb.service network-online.target
Requires=logos-radio-xvfb.service

[Service]
Type=simple
WorkingDirectory=/home/YOU
ExecStart=/home/YOU/radio-station/run-app.sh
Restart=always
RestartSec=10
TimeoutStopSec=25

[Install]
WantedBy=default.target
```

### 5.4 Unit 3 — the Liquidsoap feed `~/.config/systemd/user/logos-radio-psr.service`
The `path`/`pass` come straight from `station.json` (§4). This is the real PSR feed unit:
```ini
[Unit]
Description=Parallel Society Radio - liquidsoap feed -> radio_module MediaMTX
After=logos-radio-app.service
Wants=logos-radio-app.service

[Service]
Environment=PSR_RTMP_PATH=REPLACE_WITH_station.json_path
Environment=PSR_RTMP_PASS=REPLACE_WITH_station.json_streamKey
ExecStart=/usr/bin/liquidsoap /mnt/music/parallel-society-radio/station.liq
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
```
> **Single-feeder rule.** Only one process may push to the RTMP path. If you ever used the old looped
> ffmpeg `feed.sh`, **disable it** (`systemctl --user disable --now logos-radio-feed.service`) so it
> can't fight Liquidsoap for the same path.

### 5.5 Enable, and make it survive **reboot** (not just logout)
```bash
systemctl --user daemon-reload
systemctl --user enable --now logos-radio-xvfb.service
systemctl --user enable --now logos-radio-app.service
systemctl --user enable --now logos-radio-psr.service

# CRITICAL: without lingering, --user services stop when you log out / on boot before login.
sudo loginctl enable-linger "$USER"

# watch it come up
journalctl --user -u logos-radio-app.service -f
```
After a reboot: xvfb → app (waits ~80 s, opens panel, **auto-resumes** the station) → liquidsoap
connects → 🔴 Live, announcing. No human in the loop.

## 6. Verify end-to-end

**On the box** (no audio needed):
```bash
pgrep -x mediamtx && echo "origin up"
pgrep -x liquidsoap && echo "feed up"
curl -s "http://127.0.0.1:9997/v3/paths/list" | head    # all MediaMTX paths (status API, RADIO_API_PORT)
# Booth itself polls the path-specific endpoint the same way (radio_plugin.cpp:527):
#   curl -s "http://127.0.0.1:9997/v3/paths/get/<your-station-path>"   # 200 = receiving, else waiting
cat /mnt/music/parallel-society-radio/nowplaying.txt     # should show "artist — title", changing
tail -f ~/radio-station/app.log                          # look for announce + auto-resume lines
```

**From a listener (the real test — needs a human / Receiver):** open the **Receiver** module on another
machine → your station appears in the directory (announced every **15 s**, expires after **45 s** if the
box goes silent) → tap to play → its **signature verifies** (station = its public key). Check loudness is
even across a **talk → DJ transition** and that playback survives past the 45 s TTL. This last step is
wetware — it needs ears on the actual stream.

## 7. What persists, what doesn't

| Thing | Survives restart? | Mechanism |
|-------|-------------------|-----------|
| Station identity + config | ✅ | `station.json` (`radio_plugin.cpp:375`), respects `XDG_DATA_HOME` |
| **Autogen** station broadcast | ✅ auto-resumes | `running:true` + `keySource:autogen` → re-spawn + re-announce (`:431`) |
| **Keycard** station broadcast | ❌ **stays stopped** | key can't re-derive at boot; would broadcast unsigned → refuses (`:447`). Reconnect card + Start. |
| A **deliberately Stopped** station | ❌ (by design) | Stop writes `running:false`; identity kept, no auto-resume (`:411`, #17) |
| The app / feed processes | ✅ | `systemd --user` `Restart=` + `loginctl enable-linger` |

**Takeaway for unattended boxes:** use **Autogenerated** identity. Keycard is for a human-attended
broadcaster who reconnects the card and presses Start.

## 8. Troubleshooting & ops

| Symptom | Fix |
|---------|-----|
| Booth stuck **Waiting for OBS…** | Wrong RTMP path/pass. `path` = before `?`; `pass` = the `pass=` value in the card / `station.json`. Different machine feeding? use the box's LAN IP, not `127.0.0.1`. |
| **No `%ffmpeg`/AAC** in Liquidsoap | apt build lacks the encoder → install the static build from liquidsoap.info ([`CONNECTING-LIQUIDSOAP.md`](CONNECTING-LIQUIDSOAP.md) §1). |
| Station **not in the directory** | It only announces while **Live**; check `pgrep mediamtx` and that the feed is connected. Announce is heartbeat-only (no history) — a fresh Receiver sees you within 15 s. |
| Listeners get **silence** over onion | Tor HS descriptor can go dark (flaky HSDir uploads). A station **restart republishes**. Tracked in **#38** (reachability) / **#46** (descriptor monitor + auto-republish). |
| Loudness jumps talk↔music | Your offline normalization (§2.3) didn't cover a folder, or ReplayGain tags aren't applied — uncomment `enable_replaygain` in `station.liq`. |
| `--user` services died after logout/reboot | You skipped `loginctl enable-linger "$USER"` (§5.5). |
| Coordinate clicks miss (panel/Start) | Build/resolution differs — VNC into `:99`, find real coords, update `run-app.sh` (§5.1). |

**Ports** (all env-overridable — `radio_plugin.cpp:162-166`): `1935` RTMP in · `8889` WHIP in ·
`8890` SRT in · `8888` HLS out · `9997` status API. Tor maps HS `:80 → 8888` (`:916`).
