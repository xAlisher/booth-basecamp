# Station identity — announce v:2 signing contract (Stage 1, epic radio#24 / receiver#13/#46)

**Both radio_module (sign) and receiver_ui (verify) MUST implement this byte-for-byte identically,
or every signature fails.** This doc is the shared contract.

## Three key-source tiers (radio_ui dropdown)
1. **Anonymous (no keys)** — announce stays **v:1 unsigned** (today's behavior). No fingerprint; shows
   `IP hidden by Tor`. Truly anonymous, cannot be pinned/verified. The honest "no identity" option.
2. **Autogenerated** — v:2 signed with a device-local persistent keypair (below). Pseudonymous, pinnable,
   device-bound.
3. **Keycard (`bc:radio`)** — v:2 signed, portable/hardware-backed (Stage 3). Same signing path as autogen.

**Default (open call — leaning Autogenerated):** identity-by-default makes pinning work out of the box; a
privacy-max streamer can pick Anonymous to opt out. (Alternative: default Anonymous to preserve today's
behavior + make identity opt-in.) Persist the chosen tier in station.json.

## Keypair (autogen tier)
- secp256k1. Privkey = 32 random bytes (`secp256k1_ec_seckey_verify`); persist hex (0600) in the
  per-profile identity dir, reused across restarts (like the #17 stream identity). Regenerate only on
  explicit "new identity".
- Pubkey = 33-byte COMPRESSED (`secp256k1_ec_pubkey_create` + `_serialize` with `SECP256K1_EC_COMPRESSED`),
  emitted as lowercase hex (66 chars) in the announce `pubkey` field.
- (Stage 3 Keycard tier reuses this signing path; only the privkey source changes — `deriveKey("bc:radio")`.)

## Announce schema v:2
Add to the existing payload: `{"v":2, …existing fields…, "pubkey":"<hex33>", "sig":"<hex64>"}`.
`v:1` (unsigned) still parses for back-compat; a v:1 station is shown WITHOUT a verified identity.

## Canonical bytes to sign/verify (THE contract)
1. Build the full announce QJsonObject INCLUDING `pubkey` and `v:2` but EXCLUDING `sig`.
2. `QJsonDocument(obj).toJson(QJsonDocument::Compact)` → UTF-8 bytes.
   - QJsonObject keys are stored sorted → deterministic key order on both sides. Compact = no whitespace.
   - All numeric fields are integers (v, seq, startedAt) → no float formatting ambiguity.
3. digest = SHA-256(those bytes)  (`QCryptographicHash::Sha256`).
4. sig = `secp256k1_ecdsa_sign(digest)` → 64-byte compact → lowercase hex in `sig`.

Receiver: parse announce → remove `sig` key → re-serialize the remaining object with the SAME
`toJson(Compact)` → SHA-256 → `secp256k1_ecdsa_verify(sig, digest, pubkey)`. pubkey is IN the signed
object, so it's bound to the announce (can't swap the key). Reject on verify-fail or malformed.

## Fingerprint (display identity)
- `fp_bytes = SHA-256(pubkey_33_bytes)`; map bytes → PGP biometric word list (256 even + 256 odd),
  alternating even(byte0)/odd(byte1)/even(byte2)… Use the first **3 bytes → 3 words** (per Alisher's
  example `cowbell bottomless pancake`). The pin/verify anchor is the full pubkey; the words are the
  human recognition check.
- The 512-word table is compiled into BOTH modules (shared source) — identical or fingerprints diverge.

## Host line copy (replaces "Anonymous over Tor")
For an onion station the secondary line becomes:
- verified identity → `IP hidden by Tor · <word0 word1 word2>`
- no identity (v:1 / unverified) → `IP hidden by Tor`
Applies to BOTH the list row and the player bar (receiver `hostLine()` / `playingHostLine()`), and radio_ui
shows its own `IP hidden by Tor · <words>` self-identity in the card. Direct (non-onion) stations keep
`<host> · <privacy>` + the words when verified.

**Rationale:** a persistent fingerprint makes a station **pseudonymous, not anonymous** — so "IP hidden by
Tor" is the honest framing (the IP is hidden; the identity is a stable pseudonym), not "Anonymous".

## Security notes
- Verify lives in C++ (QML sandbox can't). Reject-then-drop invalid sigs; never render an unverified
  pubkey as if verified.
- Privkey never leaves the backend, never logged, 0600 on disk. Keycard tier: derive → sign → wipe.
- TOFU caveat (documented for Stage 2 pin): the broadcaster's own fingerprint (radio_ui card) is the
  out-of-band anchor so a listener pins the right key vs a name-squatter.
